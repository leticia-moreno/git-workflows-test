# Git Workflows Test Repository

This repository is used to demonstrate various Git workflows and the capabilities of the Manus AI GitHub skill.

## Features

- **Branching Strategy**: Utilizes a feature-branch workflow.
- **Automated Syncing**: Keeps branches up-to-date with the main branch.
- **Pull Request Automation**: Streamlines the process of opening pull requests.

## Getting Started

Clone the repository:

```bash
git clone https://github.com/leticia-moreno/git-workflows-test.git
cd git-workflows-test
```

## Contributing

Contributions are welcome! Please follow the standard GitHub flow:

1.  Fork the repository.
2.  Create a new feature branch (`git checkout -b feature/your-feature-name`).
3.  Make your changes.
4.  Commit your changes (`git commit -m 'feat: Add your feature'`).
5.  Push to the branch (`git push origin feature/your-feature-name`).
6.  Open a Pull Request.
